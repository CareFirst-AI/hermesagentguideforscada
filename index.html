Section 01

What Is a Hermes Agent? 🤔

Think of Hermes Agent like a super-smart personal assistant — but instead of a human, it's a piece of software powered by Artificial Intelligence (AI). It can read emails for you, answer questions, automate repetitive tasks, browse the web, send messages, and even write reports — all by itself.

The name "Hermes" comes from the Greek messenger god ⚡ — because this agent is all about communication, speed, and delivering information between you and the digital world.

🧠 How Does It Work? (Simple Version)

You give it a goal — like "check my email and summarize important messages." The agent uses a Large Language Model (LLM) — basically a very powerful AI brain (like ChatGPT) — to understand your instructions and figure out what steps to take. Then it connects to tools (like your email, calendar, web browser) to actually do those steps automatically.

🏭 Why Is It Useful for SCADA / Industrial Engineers?
  • 📊 Automate alarm reports — automatically summarize daily SCADA alarms into readable reports
  • 📧 Email notifications — send alert emails when critical conditions are met
  • 📋 Data collection — pull data from OPC-UA, Modbus, or REST APIs and organize it
  • 📆 Shift handover logs — auto-generate shift reports from sensor data
  • 🔍 Maintenance scheduling — monitor equipment hours and suggest PM tasks
🧩 The Building Blocks of Hermes Agent
  • 🧠 LLM Brain — Claude, GPT-4, or an open-source model like Mistral/Hermes 3
  • 🔧 Tools — web search, file reading, email, calculator, database queries
  • 💾 Memory — ability to remember past conversations or data
  • 🔁 Loop — the agent keeps working until the task is done (Reason → Act → Observe loop)
💡
Fun fact! Hermes 3 is a popular open-source AI model trained by NousResearch specifically to be excellent at "agentic tasks" — meaning it's great at planning, tool use, and following instructions step by step. You'll be using it as the "brain" of your agent!

Section 03

Step-by-Step Setup Instructions 🚀

Follow each step carefully. Don't skip any! Think of this like commissioning a new PLC — every step matters and they must be done in order.

STEP 01

🐍 Install Python on Your Computer

What are we doing? Python is like the "operating system" for our agent. Everything else runs on top of it.

Why? The Hermes Agent is written in Python, so we need it installed first.

  1. Open your web browser and go to python.org/downloads
  2. Click the big yellow "Download Python 3.x.x" button
  3. Run the downloaded file (.exe on Windows)
  4. ⚠️ VERY IMPORTANT: On the first screen, check the box that says "Add Python to PATH" at the bottom — DO NOT skip this!
  5. Click "Install Now" and wait for it to finish
  6. Click "Close" when done

✅ Verify it worked: Open Command Prompt (search "cmd" in Windows Start menu) and type:

python --version # You should see something like: Python 3.11.5
⚠️
If you see 'python' is not recognized — you forgot to check "Add Python to PATH". Uninstall Python and redo step 4!
STEP 02

📥 Install VS Code (Your Code Editor)

What are we doing? Installing a text editor where we'll write and manage our agent's code files.

Why? Regular Notepad won't work well — VS Code gives us colors, error highlighting, and a built-in terminal.

  1. Go to code.visualstudio.com
  2. Click the big "Download for Windows" button (or Mac/Linux if applicable)
  3. Run the installer and accept all defaults — click "Next" through everything
  4. After installation, open VS Code
  5. Install the Python extension: Click the puzzle-piece icon on the left sidebar, search "Python", click Install on the one by Microsoft
💡
VS Code is free and used by millions of engineers worldwide. You'll love it once you get used to it!
STEP 03

🦙 Install Ollama (Your Local AI Engine)

What are we doing? Installing Ollama — a free tool that lets you run powerful AI models like Hermes 3 entirely on your own computer.

Why? This means no subscription, no API costs, and your data never leaves your machine — critical for SCADA environments!

  1. Go to ollama.ai
  2. Click "Download" for your operating system
  3. Run the installer — it's simple, just keep clicking Next/Install
  4. After installation, Ollama runs quietly in the background (you'll see a llama 🦙 icon in the system tray)

Now download the Hermes 3 model. Open Command Prompt and type:

ollama pull hermes3 # This downloads the Hermes 3 AI model (~4.5 GB) # It will take 5-15 minutes depending on your internet speed # You'll see a progress bar — be patient! ☕

Once downloaded, test it works:

ollama run hermes3 # The AI will start! Type "Hello!" and press Enter # To exit, type /bye and press Enter
✅
If Hermes 3 replies to "Hello!" — congratulations! You now have a fully local AI running on your machine! 🎉
STEP 04

📁 Create Your Project Folder

What are we doing? Setting up a dedicated folder for all our Hermes Agent files — like creating a new panel folder in your SCADA system.

  1. Open VS Code
  2. Go to File → Open Folder
  3. Navigate to your Documents folder, create a new folder named hermes-agent, then click "Select Folder"
  4. VS Code will ask if you trust this folder — click "Yes, I trust the authors"

Now open the built-in terminal in VS Code: go to Terminal → New Terminal from the top menu.

💡
The terminal inside VS Code works exactly like Command Prompt, but it automatically starts inside your project folder. Very convenient!
STEP 05

🏗️ Create a Virtual Environment

What are we doing? Creating an isolated Python "bubble" for our project — like a dedicated VLAN in networking.

Why? This keeps our project's tools separate from other Python programs on your computer, preventing conflicts.

In the VS Code terminal, type:

python -m venv venv # Creates a folder called "venv" — this is your isolated environment

Now activate the virtual environment:

# On Windows: venv\Scripts\activate # On Mac/Linux: source venv/bin/activate # You'll know it worked when you see (venv) at the start of the line! # Example: (venv) C:\Users\YourName\Documents\hermes-agent>
⚠️
Remember: Every time you open a new terminal session, you need to activate the virtual environment again with venv\Scripts\activate. Look for the (venv) prefix!
STEP 06

📦 Install Required Python Libraries

What are we doing? Installing the Python packages (libraries) that our Hermes Agent needs to function — like installing function blocks in a PLC.

Make sure your virtual environment is active (you see (venv)), then run:

pip install langchain langchain-community ollama requests python-dotenv # langchain = the agent framework (the "brain orchestrator") # ollama = connects Python to our local Ollama AI # requests = lets the agent browse web / call APIs # python-dotenv = safely stores secret settings (like API keys)

Wait for installation to complete. You'll see many lines scroll by — this is normal! ✅

STEP 07

🔐 Create Your Configuration File (.env)

What are we doing? Creating a secure settings file that stores configuration values — like a config.ini in SCADA.

Why? We never hardcode secrets directly in code. If you share your code later, this file stays private.

  1. In VS Code, click New File (the icon at the top of the file panel)
  2. Name it exactly: .env (yes, starting with a dot!)
  3. Paste the following content into it:
# ── Hermes Agent Configuration ────────────────────── OLLAMA_MODEL=hermes3 OLLAMA_BASE_URL=http://localhost:11434 AGENT_NAME=HermesAgent MAX_ITERATIONS=10 # Add your API keys here if needed in future: # OPENAI_API_KEY=your_key_here

Save the file with Ctrl+S (or Cmd+S on Mac).

⚠️
Never share your .env file with anyone or upload it to GitHub! It will contain sensitive API keys in the future.
STEP 08

🤖 Write Your First Hermes Agent!

What are we doing? Writing the core Python code for our Hermes Agent. This is the exciting part!

  1. In VS Code, create a new file called hermes_agent.py
  2. Copy and paste the complete code below into it:
# ═══════════════════════════════════════════════════════════ # HERMES AGENT — Your Local AI Assistant # Powered by: LangChain + Ollama + Hermes 3 # ═══════════════════════════════════════════════════════════ import os import datetime from dotenv import load_dotenv from langchain_community.llms import Ollama from langchain.agents import AgentExecutor, create_react_agent from langchain.tools import Tool from langchain.prompts import PromptTemplate # ── Load settings from .env file ────────────────────────── load_dotenv() MODEL_NAME = os.getenv("OLLAMA_MODEL", "hermes3") BASE_URL = os.getenv("OLLAMA_BASE_URL", "http://localhost:11434") AGENT_NAME = os.getenv("AGENT_NAME", "HermesAgent") # ── Connect to our local Hermes 3 AI ────────────────────── llm = Ollama(model=MODEL_NAME, base_url=BASE_URL) # ═══════════════════════════════════════════════════════════ # TOOLS — What the agent CAN DO # Each tool is like a function block in PLC programming # ═══════════════════════════════════════════════════════════ def get_current_time(input: str = "") -> str: """Returns the current date and time.""" now = datetime.datetime.now() return f"Current date and time: {now.strftime('%Y-%m-%d %H:%M:%S')}" def simple_calculator(expression: str) -> str: """Evaluates a math expression. Example: '2 + 2 * 10'""" try: result = eval(expression) return f"Result of '{expression}' = {result}" except Exception as e: return f"Calculator error: {str(e)}" def read_text_file(filename: str) -> str: """Reads content from a text file in the current folder.""" try: with open(filename.strip(), 'r') as f: content = f.read() return f"File contents:\n{content}" except FileNotFoundError: return f"File '{filename}' not found." def save_to_file(content: str) -> str: """Saves text to a file. Format: 'filename.txt|||Your content here'""" try: parts = content.split("|||") filename = parts[0].strip() text = parts[1].strip() if len(parts) > 1 else content with open(filename, 'w') as f: f.write(text) return f"✅ Successfully saved to '{filename}'" except Exception as e: return f"Save error: {str(e)}" # ── Register Tools ───────────────────────────────────────── tools = [ Tool(name="GetCurrentTime", func=get_current_time, description="Get the current date and time"), Tool(name="Calculator", func=simple_calculator, description="Calculate math expressions. Input: math expression like '100 * 1.1'"), Tool(name="ReadFile", func=read_text_file, description="Read a text file. Input: filename like 'data.txt'"), Tool(name="SaveFile", func=save_to_file, description="Save text to file. Input: 'filename.txt|||content to save'"), ] # ── Agent Prompt Template ────────────────────────────────── prompt = PromptTemplate.from_template(""" You are {agent_name}, a helpful and precise AI assistant. You have access to tools to help complete tasks. Tools available: {tools} Use this format EXACTLY: Thought: Think about what to do Action: tool_name Action Input: the input for the tool Observation: [result from tool] ... (repeat as needed) Thought: I now have the final answer Final Answer: [your complete answer] Tool names: {tool_names} Question: {input} {agent_scratchpad} """) # ── Build the Agent ──────────────────────────────────────── agent = create_react_agent(llm=llm, tools=tools, prompt=prompt) agent_executor = AgentExecutor( agent=agent, tools=tools, verbose=True, max_iterations=int(os.getenv("MAX_ITERATIONS", 10)), handle_parsing_errors=True ) # ═══════════════════════════════════════════════════════════ # MAIN LOOP — Talk to your Hermes Agent! # ═══════════════════════════════════════════════════════════ def main(): print(f"\n{'='*55}") print(f" 🪄 {AGENT_NAME} is Online") print(f" Model: {MODEL_NAME} Base: {BASE_URL}") print(f"{'='*55}") print(" Type your task and press Enter.") print(" Type 'quit' or 'exit' to stop.\n") while True: user_input = input("You 👤: ").strip() if not user_input: continue if user_input.lower() in ['quit', 'exit', 'q']: print("\n👋 Hermes Agent shutting down. Goodbye!\n") break try: response = agent_executor.invoke({ "input": user_input, "agent_name": AGENT_NAME }) print(f"\n🤖 Hermes: {response['output']}\n") except Exception as e: print(f"\n❌ Error: {e}\n") if __name__ == "__main__": main()

Save the file with Ctrl+S.

Section 05

Troubleshooting Common Problems 🔧

Something not working? Don't panic! Every engineer troubleshoots. Click each problem below to see the fix.

🔴  "python is not recognized as an internal or external command" ▼
Cause: Python was installed without adding it to PATH.

Fix:
  1. Uninstall Python from Windows Settings → Apps
  2. Re-download Python from python.org
  3. Run the installer again
  4. This time, CHECK "Add Python to PATH" on the first screen before clicking anything else!
  5. Complete the installation and try again
🔴  "ConnectionRefusedError" or "Cannot connect to Ollama" ▼
Cause: Ollama is not running in the background.

Fix:
  1. Check your system tray (bottom-right icons near the clock) for the 🦙 Ollama icon
  2. If not there, open the Ollama application from your Start Menu
  3. Wait 10 seconds for it to start up
  4. Try running your agent again
  5. If still failing, open Command Prompt and type ollama serve
🔴  "ModuleNotFoundError: No module named 'langchain'" ▼
Cause: Either the virtual environment is not activated, or the pip install failed.

Fix:
  1. Check for (venv) at the start of your terminal line. If missing, activate it: venv\Scripts\activate
  2. Then reinstall: pip install langchain langchain-community ollama requests python-dotenv
  3. Run your script again
🔴  Hermes 3 gives very slow or incorrect responses ▼
Cause: Your computer may not have enough RAM, or the model is overloaded. Hermes 3 works best with 16GB+ RAM.

Fix Options:
  • Close other heavy applications (browser tabs, other software) while running the agent
  • Try a smaller model: ollama pull hermes3:3b (smaller but faster version)
  • Or try: ollama pull mistral as a lighter alternative
  • In your .env file, change OLLAMA_MODEL=hermes3 to OLLAMA_MODEL=hermes3:3b
🔴  Agent keeps looping and never gives a Final Answer ▼
Cause: The agent is confused about how to format its response, or the task is too complex.

Fix:
  1. Press Ctrl+C to stop the loop
  2. Try rephrasing your question more simply and clearly
  3. Break complex tasks into smaller steps
  4. In your .env file, reduce MAX_ITERATIONS=5 to force it to stop sooner
🔴  pip install fails with "Permission denied" or network errors ▼
Cause A (Permission): Run VS Code or Command Prompt as Administrator (right-click → "Run as administrator").

Cause B (Corporate Network/Firewall): Your company firewall may block pip. Try:
pip install --trusted-host pypi.org --trusted-host files.pythonhosted.org langchain
Or contact your IT department and ask them to whitelist pypi.org and files.pythonhosted.org.
🔴  "hermes3 model not found" when running Ollama ▼
Cause: The model hasn't been downloaded yet, or the download failed.

Fix:
  1. Make sure Ollama is running (🦙 in system tray)
  2. Open Command Prompt and type: ollama pull hermes3
  3. Wait for the full download to complete (progress bar should reach 100%)
  4. Verify it downloaded: ollama list — you should see hermes3 in the list
💡
General Rule: When in doubt, restart everything — close VS Code, stop Ollama, reopen VS Code, restart Ollama, activate your venv, and try again. Fixes 80% of issues! Classic engineering: turn it off and back on again. 😄
